# v1.0.0

This release summarizes changes since `v0.5.4`.

## New Features

- 新增智能五角色调度 `firefly-custom`，支持主 Agent、路由 Agent、审查 Agent、执行 Agent 和门禁 Agent 串行协作。
- 新增加权路由决策，综合最近用户意图、关键词、Agent 能力、历史表现和记忆偏好生成可解释的 `route:decision`。
- 新增高风险门禁审批流程，执行类风险会弹出审批卡片，说明请求内容并让用户选择继续或停止。
- 新增 Windows 系统通知，高风险审批等待用户选择时会发出桌面提醒。
- 新增自定义调度模板与持久化保存，调度步骤、角色、依赖关系和 Agent 选择可保留到后续会话。
- 新增长期记忆候选机制，支持偏好、项目、风格、决策、纠正和导入对话等分类，并支持审批、编辑、禁用和删除。
- 新增记忆检索加权，按相关性、更新时间、可信度、标签和 token 预算选择上下文，不再把全部记忆塞进提示词。
- 新增使用统计请求明细与定价能力，支持输入/输出/cache token、真实/估算拆分、供应商/模型排行和手动价格规则。
- 新增快捷键设置，支持工作台、设置、MCP、记忆、Git、写作和 Composer 等常用入口的键盘操作。
- 新增代码路径点击菜单，可从回答中的文件路径打开默认编辑器或在文件管理器中定位。
- 新增 Skill 本地扫描与统一管理入口，支持刷新本地 Skill 文件夹并按 Agent 安装使用。
- 新增 MCP 本地配置扫描增强，支持读取更多本地配置来源并执行 stdio JSON-RPC 初始化测试。

## Fixes

- 修复智能五角色执行顺序不符合预期的问题，现在按路由、主 Agent、审查、执行、门禁逐步交接。
- 修复高风险结果直接拦截的问题，现在改为要求用户审批后再继续或停止。
- 修复审批卡片缺少请求内容说明的问题。
- 修复自定义调度修改无法持久化的问题。
- 修复智能角色无法独立选择 Agent 的问题。
- 修复路由 Agent 可能接收主 Agent 输出上下文的问题，现在仅使用最近用户消息和当前输入做路由判断。
- 修复普通回答中暴露内部路由、审查、执行或门禁过程内容的问题。
- 修复使用统计只显示总 token、缺少输入/输出/cache 和成本拆分的问题。
- 修复本地 CLI 无 usage 时估算 token 与真实 usage 可能重复统计的问题。
- 修复 MCP stdio server 只跑 `--help` 导致测试结果不准确的问题，现在通过 initialize 请求验证可用性。
- 修复记忆库容易记录随意测试语句的问题，新增候选状态和精简筛选后再进入可用记忆。
- 修复文件路径右键菜单不可用的问题，补齐编辑器打开和文件夹定位动作。
- 修复部分深色主题下审批、统计、记忆、Skill 和 MCP 页面文本对比度不足的问题。

## Performance Improvements

- 优化运行事件保存和流式输出刷新，降低长回答和多 Agent 调度时的界面卡顿。
- 优化使用统计聚合，减少运行中重复扫描和重复计算。
- 优化长期记忆检索，按预算裁剪并优先使用精华摘要。
- 优化 MCP 测试超时与进程清理，避免异常 server 长时间占用。
- 优化设置页和工作台样式覆盖，减少深浅色切换后的重排和闪烁。
